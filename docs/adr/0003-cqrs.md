# ADR-0003 — CQRS com MediatR

## Status

Aceito

## Contexto

Operações de leitura e escrita possuem responsabilidades distintas.

## Decisão

Adotar CQRS utilizando MediatR.

Commands modificam o estado da aplicação.

Queries realizam apenas consultas.

Cada operação possui seu próprio Handler.

## Consequências

### Positivas

- Separação clara entre leitura e escrita
- Melhor organização
- Fácil evolução
- Alta testabilidade

### Negativas

- Maior número de classes
- Mais arquivos para manter

## Alternativas consideradas

Services tradicionais contendo métodos de leitura e escrita na mesma classe.
