# Contribuindo

Obrigado pelo interesse em contribuir.

---

# Fluxo

1. Fork do projeto
2. Criar uma Branch
3. Implementar alteração
4. Executar testes
5. Abrir Pull Request

---

# Pull Requests

Todo Pull Request deve:

- Compilar
- Passar nos testes
- Não reduzir cobertura
- Seguir o padrão de código

---

# Commits

Utilizamos Conventional Commits.

Exemplos:

```
feat:

fix:

docs:

refactor:

test:

chore:
```
